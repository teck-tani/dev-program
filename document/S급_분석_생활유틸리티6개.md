## 1. 비밀번호 생성기 (현재 A-)

### 잘 된 것
- Web Crypto API 보안 생성
- 엔트로피 기반 강도 표시 (4단계)
- 최대 10개 동시 생성
- Fisher-Yates 셔플 (편향 없는 랜덤)
- 히스토리 50개 보관
- 대/소문자/숫자/특수문자 토글

### S급에 부족한 것

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| **필수** | **패스프레이즈 모드** | "correct-horse-battery-staple" 형태 단어 조합 비밀번호. 1Password/Bitwarden 기본 제공. 한국어 단어 리스트 넣으면 독보적 차별화 |
| **필수** | **모호한 문자 제외** 옵션 | `0O1lI` 같은 헷갈리는 문자 제외 체크박스. LastPass/Dashlane 기본 기능 |
| **필수** | **해독 소요시간 표시** | "이 비밀번호를 푸는 데 약 3억 년" — 엔트로피보다 직관적 |
| 높음 | 프리셋 규칙 | "은행용(숫자+영문 8~16자)", "WiFi용", "PIN 4자리" 원클릭 |
| 높음 | 강도 상세 피드백 | "대문자를 추가하면 Strong이 됩니다" 같은 구체적 개선 제안 |
| 중간 | CSV/텍스트 다운로드 | 대량 생성 시 파일로 저장 |
| 중간 | 커스텀 특수문자 세트 | 사용자가 포함/제외할 특수문자 직접 지정 |
| 낮음 | Have I Been Pwned 연동 | 생성된 비밀번호가 유출DB에 있는지 확인 (API) |

### 구현 체크리스트
- [ ] 패스프레이즈 탭/모드 추가 (한국어+영어 단어 리스트)
- [ ] `excludeAmbiguous` 옵션 추가 → CHAR_SETS에서 0O1lI 제거
- [ ] 해독 소요시간 계산 함수 (entropy → 초/분/시/년 변환)
- [ ] 프리셋 버튼 (은행/WiFi/PIN/일반)
- [ ] 강도별 개선 제안 메시지

## 2. 랜덤 생성기 (현재 A-)

### 잘 된 것
- 5탭 (숫자/이름/색상/주사위/동전)
- D4~D20 다이스 + D6 점 표시
- 동전 뒤집기 애니메이션 + 히스토리
- 색상 HEX/RGB/HSL 동시 표시
- 한국어/영어 이름 생성
- 정수/소수 선택, 범위/개수 지정

### S급에 부족한 것

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| **필수** | **리스트 셔플 탭** | 목록 붙여넣기 → 랜덤 순서 정렬. wheelofnames.com 핵심. 제비뽑기/순서정하기 검색량 높음 |
| **필수** | **팀 나누기 탭** | 이름 입력 → N팀으로 랜덤 분배. 학교/회사에서 폭발적 사용량 |
| 높음 | 숫자 프리셋 | "로또(1~45, 6개 중복없이)", "OTP(6자리)", "좌석번호" 원클릭 |
| 높음 | RPG 주사위 표기법 | `3d6+5` 같은 입력. D&D/TRPG 유저 유입 |
| 높음 | 색상 팔레트 모드 | 보색/유사색/단색 팔레트 자동 생성. 디자이너 타겟 |
| 중간 | 이름 성별 구분 | 남자/여자 이름 분리 생성 |
| 중간 | 숫자/이름 탭 히스토리 | 동전만 히스토리 있음 → 다른 탭에도 추가 |
| 중간 | 결과 내보내기 | CSV/JSON으로 다운로드 |
| 낮음 | 중복 제거 옵션 | 숫자 생성 시 "중복 허용/금지" 토글 |

### 구현 체크리스트
- [ ] "셔플" 탭 추가 (textarea → 줄 단위 셔플)
- [ ] "팀" 탭 추가 (이름 입력 + 팀 수 → 랜덤 배분)
- [ ] 숫자 탭에 프리셋 버튼 (로또/OTP 등)
- [ ] 숫자 탭에 "중복 없이" 체크박스
- [ ] RPG 표기법 입력 (`NdS+M` 파서)
- [ ] 색상 팔레트 모드 (보색/유사색 알고리즘)
- [ ] 모든 탭에 히스토리 추가

## 3. IP 주소 조회 (현재 A)

### 잘 된 것
- IPv4/IPv6 자동 감지
- 커스텀 IP/도메인 조회
- VPN/프록시/호스팅 감지 뱃지
- OpenStreetMap 지도 연동
- 위치정보 (국가/지역/도시/좌표/타임존)
- 네트워크 정보 (ISP/조직/ASN/Reverse DNS)
- User-Agent 파싱 (브라우저/OS/화면/언어/쿠키/DNT)
- 국기 이모지

### S급에 부족한 것

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| 높음 | **인터넷 속도 테스트** | 다운/업로드 속도 측정. whatismyip.com 핵심. 검색량 매우 높음. 체류시간 폭증 |
| 높음 | **DNS Leak 테스트** | VPN 사용자 타겟. DNS 요청이 VPN 밖으로 새는지 확인 |
| 높음 | **WebRTC Leak 체크** | VPN 쓰더라도 WebRTC로 실제 IP 노출되는지 확인 |
| 중간 | Whois 조회 | 도메인 입력 시 등록 정보 표시 |
| 중간 | 결과 이미지 저장 | "내 IP 정보" 카드를 이미지로 다운로드 |
| 중간 | HTTP 헤더 표시 | 요청 헤더 정보 표시 (Accept, Encoding 등) |
| 낮음 | IP 히스토리 | 이전 조회 기록 localStorage 저장 |
| 낮음 | 포트 스캔 | 열린 포트 확인 (서버 측 필요) |

### 구현 체크리스트
- [ ] 속도 테스트 섹션 (난이도 높음 — CDN 서버 또는 서비스 필요)
- [ ] WebRTC Leak 감지 (`RTCPeerConnection` → 로컬 IP 추출)
- [ ] DNS Leak 테스트 (고유 서브도메인 쿼리 → DNS 서버 확인)
- [ ] HTTP 요청 헤더 표시 섹션
- [ ] 결과 이미지 내보내기 (html2canvas)

### 참고
- 속도 테스트는 구현 난이도가 높아 별도 검토 필요
- WebRTC Leak은 비교적 간단하게 구현 가능 (RTCPeerConnection API)

## 4. 글자수 세기 (현재 A+)

### 잘 된 것
- 공백 포함/제외 글자수
- 단어/줄/문단/바이트 수
- 한/영/숫자/특수문자 구분 카운트
- 읽기 시간 (한국어 500자/분, 영어 200단어/분)
- 200자/400자 원고지 매수
- SNS 제한 프로그레스바 6종 (트위터/인스타/유튜브/페이스북/블로그)
- 키워드 빈도 분석 TOP 10
- 텍스트 변환 5종 (대문자/소문자/첫글자/공백제거/트림)

### S급에 부족한 것

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| 높음 | **문장 수 카운트** | 마침표/물음표/느낌표 기준 문장 수. 글쓰기 도구 기본 |
| 높음 | **파일 업로드** | .txt/.docx 드래그&드롭으로 분석. 타이핑 없이 바로 분석 |
| 중간 | 네이버 블로그/카페 제한 추가 | SNS 제한에 네이버(65,536자), 카카오톡(5,000자) 등 한국 플랫폼 추가 |
| 중간 | 중복 문장 감지 | 같은 문장 반복 하이라이트. 카피킬러 느낌 |
| 중간 | 가독성 점수 | Flesch-Kincaid 같은 가독성 지표 (한국어 버전) |
| 낮음 | 맞춤법 검사 연동 | 한국어 맞춤법 API 연동 (구현 난이도 높음) |
| 낮음 | 텍스트 비교 | 이전/이후 텍스트 변경사항 카운트 |

### 구현 체크리스트
- [ ] 문장 수 카운트 (정규식: `/[.?!。！？]+/g` 기준)
- [ ] 파일 업로드 (FileReader API, .txt 지원)
- [ ] SNS 목록에 네이버 블로그/카페, 카카오톡, 인스타 캡션 추가
- [ ] 중복 문장 감지 & 하이라이트

### 참고
- 이미 거의 S급 수준. 소폭 개선만으로 충분

## 5. 궁합 테스트 (현재 A-)

### 잘 된 것
- 3탭 (띠 궁합/별자리 궁합/MBTI 궁합)
- 삼합/육합/상충 정확한 데이터
- 원소(불/흙/바람/물) 기반 별자리 궁합
- MBTI 골든페어 매칭
- 애니메이션 게이지 (원형/프로그레스바)
- 참조 테이블 (삼합표, 원소표, 골든페어표)

### S급에 부족한 것

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| **필수** | **혈액형 궁합 탭** | A/B/AB/O 궁합. 한국/일본 문화에서 필수. 이거 없으면 한국 시장에서 불완전 |
| **필수** | **결과 공유** | 카카오톡/SNS 공유 버튼 + 결과 이미지 생성. 궁합은 바이럴의 핵심 콘텐츠 |
| 높음 | 복합 궁합 점수 | 띠+별자리+MBTI+혈액형 통합 점수 (가중 평균). "종합 궁합 85점!" |
| 높음 | 상세 리포트 확장 | 현재 1~2문장 → "장점/주의점/성공 팁" 3~5문단으로 확장 |
| 높음 | 생년월일 자동 감지 | 년도 입력 → "당신은 용띠입니다!" + 별자리 자동 계산 |
| 중간 | 궁합 순위 | "당신과 가장 잘 맞는 띠 TOP 3" 표시 |
| 중간 | 유명인 비유 | "당신의 궁합은 [유명 커플]과 비슷합니다" |
| 중간 | 궁합별 이모지 반응 | 90점+ 하트 폭발, 50점- 비 오는 이모지 등 |
| 낮음 | 관계 유형별 점수 | 연애/우정/직장 궁합 따로 표시 |

### 구현 체크리스트
- [ ] 혈액형 궁합 탭 추가 (4x4 궁합 매트릭스)
- [ ] 카카오 공유 SDK 연동 (Kakao.Link.sendDefault)
- [ ] 결과 이미지 생성 (html2canvas → 다운로드/공유)
- [ ] 복합 점수 계산 로직 (가중 평균)
- [ ] 상세 리포트 텍스트 확장 (번역 키 추가)
- [ ] 생년월일 입력 → 띠+별자리 자동 계산 UI

### 참고
- **혈액형 + 카카오 공유 = 바이럴 트래픽 폭증 가능성 가장 높은 도구**
- 궁합 테스트는 검색량이 높고 공유율이 높은 콘텐츠

## 6. 영업일 계산기 (현재 A-)

### 잘 된 것
- 3모드 (영업일 더하기/빼기, 두 날짜 사이 영업일, D-day 영업일)
- 한국 공휴일 2025~2027 하드코딩 (대체공휴일 포함)
- 근로자의 날 포함/제외 토글
- 건너뛴 주말·공휴일 상세 표시
- 다가오는 공휴일 목록

### S급에 부족한 것

| 우선순위 | 기능 | 설명 |
|---------|------|------|
| **필수** | **사용자 정의 휴일 추가** | 회사 창립기념일, 연차 등 커스텀 휴일 관리 (localStorage). 모든 경쟁사 기본 |
| **필수** | **달력 뷰** | 영업일/주말/공휴일 색상 구분 달력. 시각적으로 한눈에 파악 |
| 높음 | 2028년 이후 공휴일 | 현재 2027까지만 → 음력 알고리즘으로 동적 계산 |
| 높음 | 마감일 저장 | 여러 마감일을 라벨과 함께 localStorage 저장 |
| 높음 | 구글 캘린더 추가 | "계산 결과를 캘린더에 추가" 버튼 |
| 중간 | 해외 공휴일 프리셋 | 미국/일본/중국 등 선택 → 글로벌 확장 |
| 중간 | 반복 계산 | "매월 셋째 금요일", "격주 월요일" 등 |
| 중간 | 결과 복사/공유 | "2026-03-15 (화) — 영업일 10일 후" 텍스트 복사 |
| 낮음 | 커스텀 주말 설정 | 금~토 (중동), 목~금 등 다른 주말 패턴 |

### 구현 체크리스트
- [ ] 사용자 정의 휴일 관리 UI (추가/삭제, localStorage 저장)
- [ ] 미니 달력 컴포넌트 (영업일=파랑, 주말=회색, 공휴일=빨강)
- [ ] 음력 공휴일 동적 계산 (설날, 추석, 부처님오신날)
- [ ] 마감일 저장 기능 (라벨 + 날짜 + D-Day 대시보드)
- [ ] 구글 캘린더 링크 생성 (`calendar.google.com/calendar/render?action=TEMPLATE&...`)

## 종합 임팩트 순위

| 순위 | 도구 | 핵심 개선 | 예상 효과 |
|------|------|----------|----------|
| **1** | **궁합 테스트** | 혈액형 탭 + 카카오 공유 | 바이럴 트래픽 폭증 가능성 가장 높음 |
| **2** | **랜덤 생성기** | 리스트 셔플 + 팀 나누기 | 학교/회사 유입, 검색량 높음 |
| **3** | **비밀번호 생성기** | 패스프레이즈 + 해독시간 | 경쟁사 대비 확실한 차별화 |
| **4** | **영업일 계산기** | 커스텀 휴일 + 달력 뷰 | 직장인 필수 도구화 |
| **5** | **IP 주소** | 속도 테스트 (WebRTC Leak 먼저) | 검색량 최고지만 구현 난이도 높음 |
| **6** | **글자수 세기** | 파일 업로드 + 문장 수 | 이미 거의 완성, 소폭 개선 |
