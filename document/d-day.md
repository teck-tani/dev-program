## 왜 S급 가능성이 높은가?

### 1. 시장 공백이 크다
- TheDayBefore(4000만 다운로드)가 앱으로만 존재 → **웹에서 이 수준의 D-Day 도구가 없음**
- timeanddate.com은 기능이 4개 페이지에 분산 → 통합 도구 부재
- superkts.com은 단일 D-Day만 (복수 관리 X)

### 2. 한국 시장 킬러 피처 3개가 웹에 없음
- **음력 지원**: 부모님 음력 생신이 매년 양력 날짜가 달라짐 → 자동 계산 가치 극대
- **커플 기념일**: 100일/200일/1000일 + 매월 14일 기념일 자동 생성 → 한국 특화
- **수능/시험 D-Day 프리셋**: 매년 수능일 원클릭 → 학생 트래픽 폭발

### 3. 아무도 안 하는 것: 진행률 프로그레스 바
- 시작일~목표일 사이에서 "지금 65% 지점" → 동기부여 시각화
- 검색해봐도 이 기능을 제공하는 D-Day 도구가 **단 하나도 없음**

## 기능 상세

### 다중 단위 표시
- 현재: `D-30` (일수만)
- 목표: `30일 = 4주 2일 = 720시간 = 43,200분 = 2,592,000초`
- 년+월+일 복합 표시 (예: 2년 3개월 15일)

### 기념일 마일스톤 자동 생성
- 기준일 입력 시 주요 마일스톤 목록 자동 생성:
  - **일수 기반**: 100일, 200일, 300일, 365일, 500일, 1000일, 2000일, 10000일
  - **연 기반**: 1주년, 2주년, 3주년...10주년
  - 각 마일스톤의 **실제 날짜 + 요일** 표시
  - 지난 것 vs 다가오는 것 시각적 구분

### 진행률 프로그레스 바
- 시작일 설정 (선택사항) → 목표일까지의 진행률을 % 표시
- 예: 수능까지 D-100 / 전체 준비기간의 65% 경과
- 시각적 동기부여 요소

### 커플 기념일 모드
- 사귄 날 입력 시 자동 생성:
  - 100일, 200일, 300일, 400일, 500일, 1000일, 2000일, 10000일
  - 매월 14일 기념일: 다이어리데이(1/14), 발렌타인데이(2/14), 화이트데이(3/14), 블랙데이(4/14), 로즈데이(5/14), 키스데이(6/14), 실버데이(7/14), 그린데이(8/14), 뮤직데이(9/14), 와인데이(10/14), 무비데이(11/14), 허그데이(12/14)

### 음력 지원
- 양력 ↔ 음력 변환
- 음력 기념일 입력 → 매년 양력 날짜 자동 계산
- 한국 시장에서 최강 차별화 (웹 도구 중 미지원)

### 프리셋 날짜
- **시험**: 수능(매년 11월 셋째 목요일), 공무원시험, 자격증시험
- **공휴일**: 새해(1/1), 설날(음력), 추석(음력), 크리스마스(12/25)
- **특별일**: 어린이날, 어버이날, 스승의날

## 결론

> **현재 B급, S급 포텐셜은 최상급.**
>
> 기본 기능(복수 관리, 카테고리, 실시간 카운트다운)은 잘 되어 있지만,
> **기념일 마일스톤 + 커플 모드 + 음력 + 진행률 바** 4가지가 빠져서
> 한국 시장에서의 킬러 포인트가 없음.
>
> 이 4가지만 추가하면 **웹 D-Day 도구 중에서 경쟁자가 거의 없는** 상황.
>
> 특히 **기념일 마일스톤 자동 생성**과 **진행률 프로그레스 바**는
> 노력 대비 효과가 가장 크므로 최우선 구현 추천.
